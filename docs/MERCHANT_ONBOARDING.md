# 加盟店オンボーディング手順（1店舗ごとの完全チェックリスト）

> ⚠️ **【2026-07-24 方針転換】加盟＝無料スタート＋有料オプション制。** 基本掲載は**無料**になったので、下記手順のうち「決済（Stripeリンク送付／初回課金）」「初期登録料」「先行割引」「3か月縛り」は**基本フローから外れる**（旧モデル・凍結）。新フロー＝①申請 → ②審査・承認 → ③ピン/カード制作 → ④**無料で掲載開始**＋PIN送付。有料オプション（①イベント企画 ②スポンサー表記 ③上位表示・特集掲載 ④追加クーポン枠）を希望する店だけ、内容に応じて個別見積り→カード(Stripe)/振込で前払い→提供。Stripe決済リンクは有料オプション用に温存。

**新しい加盟店が決まったら、この1枚を上から順にやる。** 迷ったらここに戻る。
（各機能の詳細：料金＝[MERCHANT_APPLICATION_FORM.md] ／ 決済＝[STRIPE_SETUP.md] ／ 混雑ランプ＝[CROWD_MONITOR.md] ／ 規約＝[LEGAL_DRAFTS.md]）

---

## STEP 1｜申請を受ける・審査する

- [ ] 申請フォーム（https://tally.so/r/GxyO8L ）の通知メールを受信
- [ ] 内容を確認（掲載内容・特典が適正か／写真は使えるか／実店舗か）
- [ ] **キャンペーンの残枠を確認**（初期登録料：先着10店＝無料 → 11〜30店＝50%OFF → 31〜50店＝30%OFF）
      ※ Stripeの「クーポン」画面で `OPENING100` などの**引き換え回数**を見れば残数がわかる（例：3/10）

## STEP 2｜承認メールを送る（料金・支払い方法の案内）

- [ ] プランと金額（**すべて税込**）を明記
      - ライト ¥3,000／スタンダード ¥6,000／プレミアム ¥12,000（月額）
      - 初期登録料 ¥5,000（**キャンペーン適用後の金額**を明記）
- [ ] **最低利用期間3か月**を明記（途中解約でも残余期間の支払いが必要）
- [ ] 支払い方法に応じて：
  - **カード** … Stripeの**決済リンク**を貼る ＋ **クーポンコード**（`OPENING100` / `FOUNDER50` / `EARLY30`）を伝える
  - **銀行振込** … 請求書を作って送る
    ```
    node docs/merchant/make_invoice.js --shop "○○食堂" --plan standard --months 3 --init 0
    ```
    （3か月分前払いが基本／12か月一括なら1か月分サービス／振込手数料は店負担）
- [ ] 利用規約・特商法表記の場所を案内

## STEP 3｜入金を確認する

- [ ] **カード**：Stripeダッシュボードで課金成功を確認
- [ ] **振込**：着金を確認（**入金確認まで掲載作業を始めない**）

## STEP 4｜掲載データを作る（`data/shops.json`）

- [ ] 店舗ID（`id`）を決める（英数小文字。例：`kakigawatei`）← **後の工程で何度も使う**
- [ ] 住所から**緯度経度**を取得して `lat` / `lng` に入れる
- [ ] `name` `category` `districtId` `address` `radius`（既定120m）を記入
- [ ] `offer`（特典の見出し・説明）を記入
- [ ] `coupon`（`title` / `terms` / `staffPIN`）を記入
      - ⚠️ **`staffPIN` は店舗ごとに違う4桁にする**（現状デモは全店 `1234`。**使い回さない**）
- [ ] `links`（web / tel）

## STEP 5｜ピン・図鑑カードを制作する

- [ ] 送られてきた写真をもとに、ゲーム調のお店ピン／カードを制作
- [ ] スタンダード以上：**通常版＋豪華版（3回来店で昇格）の2枚**
- [ ] プレミアム：さらに**特別仕様の豪華カード**
- [ ] 画像を `assets/` に配置

## STEP 6｜デプロイする

- [ ] `git push origin main`（GitHub Pages ＝ Web版に即反映）
- [ ] ⚠️ **iOSアプリには自動で反映されない**（www を同梱しているため）。**次のiOSビルド時にまとめて反映**する

## STEP 7｜混雑判定器（お店の混雑ランプ）をセットアップする

> 詳細：[CROWD_MONITOR.md]。**お店に渡すURLは全店共通**：
> https://kakigawatei.github.io/nagaoka-tanken/crowd.html

- [ ] **Firebase → Authentication → ユーザー → ユーザーを追加**
      - メール：**そのお店の実在アドレス**（推奨。お店が自分でパスワード再設定できる）
      - パスワード：店ごとに違うものを設定
- [ ] 作成したユーザーの **UID をコピー**
- [ ] **Firestore → `shopOwners` コレクション**（**ルート直下**。`cards` の下などに作らないこと）
      - **ドキュメントID ＝ コピーしたUID**
      - フィールド：**`shopId`**（string）＝ STEP 4 で決めた店舗ID
      - ⚠️ **`shopId` の "d" は小文字**（`shopID` だと動かない。Firestoreは大文字小文字を区別する）
- [ ] お店に **URL・メールアドレス・パスワード** を伝える
- [ ] **「ホーム画面に追加」してもらう**と、次からワンタップで開ける
- [ ] 使い方を説明：
      - 🟢空いています／🟡やや混雑／🔴混雑しています／⚪️表示を消す のボタンを押すだけ
      - **押すと10分のカウントダウン**が始まり、**10分たつと自動で表示が消える**
      - **押し忘れても不利益はない**（表示が消えるだけ）。**忙しいときは無理に押さなくてよい**と必ず伝える

## STEP 8｜掲載完了を連絡する

- [ ] 掲載開始のメール（**入金確認後、通常7営業日以内**）
- [ ] **スタッフ用PIN**（クーポン確認用）を伝える
- [ ] クーポンの使い方を説明（お客様が「使う」→ **スタッフがPINを入力**して確定。**スクショでは使えない**）
- [ ] カード払いの店には、**Stripeの顧客ポータル**（カード変更・領収書DL）のURLを案内
      - ⚠️ **解約はポータルからできない設定**（最低利用期間3か月のため）。解約はメール申請と伝える

## STEP 9｜広報する

- [ ] **公式Instagramで加盟店紹介の投稿**（詳細：[INSTAGRAM_PLAN.md] の「型D」）
      → これが**次の営業の実績サンプル**になる（「加盟いただくとこう紹介されます」）
- [ ] プレミアムの場合：**スポンサー表記**をイベント告知等に反映

## 運営として定期的に見るもの

- **Stripe**：課金失敗（カード期限切れ）が出ていないか
- **Stripeのクーポン残数**：キャンペーン枠が埋まったら、次の割引コードに切り替える
- **混雑ランプ**：お店がちゃんと押してくれているか（押されていないなら、運用が回っていないサイン）
- **加盟店の掲載内容**：季節メニュー等で古くなっていないか

## 店舗が増えてきたら必ずやること（技術的な宿題）

- [ ] **`staffPIN` を全店で別々の値にする**（現状デモは `1234` 固定。`data/shops.json` は公開ファイルなので、PINは"クーポンの二重使用を防ぐ簡易ロック"程度の強度しかない点も理解しておく）
- [ ] 加盟店が10店を超えたら、**shops.json の管理をスプレッドシート等から生成する**仕組みを検討
- [ ] iOSアプリを**定期的にリビルド**して、Web版との差分を解消する
