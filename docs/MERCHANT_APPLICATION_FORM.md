## Stripe プラン設計（確定：2026-07-14・すべて税込）

月額サブスク。承認メールにプランごとのStripe Payment Link（月額）を貼って誘導。
**表示・請求はすべて税込**（インボイス未登録の免税事業者のため、税別＋別途課税ではなく税込総額表示に統一）。

プラン内容は **2026-07-14 masaさん指示で改訂**（クーポンをライトに降ろし、スタンダードの主役を「オリジナル図鑑カード＋リピートでグレードアップ」に、プレミアムを「店舗独自イベント」に）。

| プラン | 月額（税込） | 内容 |
|---|---|---|
| **ライト** | ¥3,000 | 地図にお店ピン掲載＋お店ページ（基本情報・写真・紹介文）＋**クーポン1種**（来店特典） |
| **スタンダード** | ¥6,000 | ライト＋**お店のオリジナル図鑑カード**（お店紹介入り）＋**3回来店で豪華版カードに昇格**（★1〜3も付く／既存の`DELUXE_VISIT_THRESHOLD=3`・`VISIT_STAR_THRESHOLDS=[3,7,15]`をそのまま流用）＋「今日の得するお店」枠に候補入り |
| **プレミアム** | ¥12,000 | スタンダード＋**お店独自のオリジナルイベント**（ヒアリング→企画→アプリのアップデートで実装・公開。※即時反映ではない）＋**クーポン複数**＋**特別仕様の豪華カード**（金枠・限定演出／イベント限定カード）＋**スポンサー表記**（イベントタイトル・クレジットに店名掲出）＋**AI活用の相談・導入サポート**（おまけ）＋上位表示＋簡易レポート |

> スタンダードの「リピートでカードがグレードアップ」は**新規実装ではなく既存のお宝カード機能の転用**：訪問3回で豪華版イラストに昇格、訪問回数で★1/2/3。加盟店カードにも同じ仕組みを適用すればよい（＝再来店の動機付けがゲーム側に既にある）。
>
> **プレミアムの運用方針（2026-07-14 masaさん）**：
> - **オリジナルイベントは「即納」ではない**。加盟店にヒアリング → 企画 → ゲームに組み込み → **バージョンアップのタイミングでリリース**。営業時も「すぐ反映ではなく、相談しながら期間限定イベントを作っていく」と正直に伝える（資料の※注記もその文言）。
> - **カードは1店ずつ手作りする**（Codex等で生成）。手間はかかるが「一つずつやるのが誠実」という方針。制作枚数が増えても**初期登録料は一律¥5,000のまま**（プラン別料金にはしない）。
> - **スポンサー表記**：イベントタイトル／クレジットに加盟店名を掲出。
> - **AI導入サポート**：AIの導入・業務活用の相談に運営が応じる“おまけ”特典（プレミアムのみ）。

- **初期登録料：¥5,000（税込・初回のみ）** — ピン/カード制作、位置設定、掲載データの初期登録。実証中に無料キャンペーンにする場合は営業資料（`docs/merchant/`）の該当ブロックを外す。
- 加盟店向け営業資料（A4・4ページPDF）＝ `docs/merchant/nagaoka_merchant_guide.pdf`（ソース：同フォルダの `merchant_guide.html`）。
- 支払い：StripeのPayment Link（月額）でカード管理・自動継続・領収書・解約までStripe側が対応 → こちら側サーバー不要。
- 契約前提：**利用規約・特商法表記・プライバシーポリシー**を用意してから募集開始（法人相手は特に必須）。

## 運営側の受付〜掲載チェックリスト（1件ごと）

1. フォーム送信のメール通知を受領
2. 内容審査（特典の適正／掲載可否／写真の質）
3. 承認メール送付（プラン確認＋Stripe Payment Link）

6. `shops.json` にレコード追加（id / name / category / lat,lng / offer / coupon.redeemPIN 等）
7. デプロイ → 掲載開始をメール連絡（スタッフ用PINも通知）
