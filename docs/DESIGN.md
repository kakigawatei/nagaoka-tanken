# ながおか探検録 — 設計書

新潟県長岡市を実際に歩いて巡る位置情報（GPS）ゲーム。史跡・名所に近づくと「お宝」を発見し、コインとXPを獲得。集めたコインで買い物、XPでレベルアップ。図鑑を埋めながら長岡の歴史・文化を知る「ご当地さんぽRPG」。

---

## 1. コンセプト
- **歩く動機を作る**：実際にその場所へ行くとGPSが反応してご褒美（お宝・コイン）。
- **長岡を知る**：お宝＝史実カード。集めるほど長岡の歴史・産業・偉人・自然に詳しくなる。
- **サーバーレス／プライバシー重視**：位置情報は端末内だけで処理し、外部送信しない。進捗は `localStorage`。
- **既存作と同じ配信手法**：GitHub Pages（HTTPS）配信。iPhone/Androidブラウザで動くPWA。

## 2. 前提・制約
- HTTPS必須（Geolocation APIはHTTPSでのみ動作）→ GitHub Pagesで満たせる。
- GPS誤差（都市部で数十m）前提の判定設計にする。
- 実地に行かずに開発・テストできる**位置シミュレーション**を最初から用意（重要）。
- 歩きスマホ抑制・私有地に立ち入らせない配慮（POIは公共からアクセス可能な地点に限定）。

## 3. 地区（GPSゾーン）＝長岡市の地域
合併後の長岡市の地域構成をそのままゾーンにする（「それぞれの地区に行くと反応」を実現）：

`長岡（中心部）／中之島／越路／三島／山古志／小国／和島／寺泊／与板／栃尾／川口`

各地域に複数のスポット（POI）を配置。地域コンプでボーナス、上位地域はレベル解放。

## 5. コアループ
1. 歩く → スポット圏内（半径の目安 **50–80m**）にGPSが入る
2. 「発見！」演出 →
   - **初回**：お宝（図鑑登録）＋大コイン＋XP
   - **再訪**：小コイン（1日1回・逓減、連続来訪ストリークで加算）
3. コインで**買い物**（探索補助・装飾・消費アイテム）
4. XPで**レベルアップ** → 新地域・機能・称号を解放

## 6. システム詳細
### 6.1 GPS／位置判定
- `navigator.geolocation.watchPosition`（`enableHighAccuracy:true`）。
- 各POIへの距離は **Haversine** で算出。閾値内で「圏内」。
- 誤差対策：数秒の滞在確認＋直近数点の平均化、半径に余裕を持たせる。
- **近接コンパス**：最寄りの未発見スポットへの方角＋距離を矢印表示（宝探し感）。
### 6.2 お宝／図鑑
- POIごとに1お宝カード（史実タイトル・時代・解説文・豆知識・写真枠）。
- 図鑑コンプ率で称号。地域別タブ。
### 6.3 経済（コイン）
- 初回発見：大（例100–300、レア度で変動）
- 再訪：小（例10、日次リセット、ストリークボーナス）
- 地域コンプボーナス、デイリーミッション
### 6.4 ショップ／買い物
- 探索補助：レーダー範囲拡大、最寄りヒント、フォトフレーム
- 装飾：アバター／スタンプ／称号枠
- 消費：お宝の再演出、ダブルコイン券
### 6.5 レベル／成長
- XP累積でレベル、レベルで地域・ショップ品・称号を解放。作業感を避けるカーブ設計。
### 6.6 画面構成
- **ホーム**：現在地・最寄りスポット・コンパス・所持コイン・レベル
- **マップ**：地域一覧 → スポット（発見済／未発見は `???` シルエット）
- **図鑑**：お宝コレクション＋解説
- **ショップ**
- **実績／称号**
- **設定**：データ書き出し／読み込み、位置精度、**開発用シミュレート**

## 7. 技術構成
- PWA（`manifest.json` + Service Worker）。単一〜少数のHTML＋JS、軽量DOM UI（＋必要箇所のみCanvas演出）。
- 保存：`localStorage`（キー接頭辞 `tekuteku_`）。書き出し／読み込みでバックアップ可。
- 地図：**MVPは自作スタイライズドマップ＋コンパス**。将来 **Leaflet + OSMタイル**で実地図（要ネット・帰属表示）＝フェーズ2以降。
- **データ分離**：`districts.json` / `pois.json`（コンテンツ）とエンジンを分離 → コンテンツ追加を安価モデルで量産可能に。
- 開発支援：位置シミュレーション（デバッグ移動・テレポート・GPSログ再生）。実地に行かずテスト可能。
- 配信：GitHub Pages（HTTPS）。

## 8. データモデル（要点）
```
district = { id, name, center:{lat,lng}, unlockLevel }
poi = {
  id, districtId, name, lat, lng, radius,
  rarity, coinsFirst, coinsRepeat, xp,
  category, treasure:{ title, era, blurb, image }
}
save(localStorage) = {
  coins, level, xp,
  found:[poiId...],
  visits:{ poiId:{ count, lastDate } },
  streak, inventory, settings
}
```
`data/pois.sample.json` にスキーマ確定用のサンプルあり。

## 9. プライバシー・安全
- 位置は端末内処理のみ、外部送信なし。GPS許可の目的説明画面を用意。
- 歩きスマホ注意喚起。POIは公共からアクセス可能な地点に限定し、私有地・危険箇所へ誘導しない。
