# 牧野家 育成×年貢×図鑑拡張システム 設計提案書

> 「ながおか探検録」の新中核システム。長岡藩主・牧野家へ年貢（おみやげ）を納めて図鑑エリアを広げる育成ゲーム。
> Fable（ゲームデザイナー役）による設計。masa（オーナー）承認待ち → Phase1から一緒に構築予定。
> 作成: 2026-07-21 ／ 継続検討時のFable agentId: `ae7bbcf379632e351`

## 1. コンセプト
発見した「おみやげ」を記念品で終わらせず、**牧野の殿への年貢**という第二の使い道にする。既存ループ（GPS発見→図鑑→コイン/XP→ガチャ）の上に「集めたおみやげを納める→殿が喜ぶ→新スポット/エリア解禁→また探検」という**もう一段上のループ**を被せる。ダブりに行き先が生まれ、「探検家としての格（称号）」とは別に**「牧野家からの信頼（家臣の出世）」という二本目の成長軸**ができ、コンプ後も遊び続ける理由になる。

## 2. 導入演出
- **トリガー**: 目安は発見数10前後（全32の約1/3）。※称号t5「郷土史コレクター」相当のタイミングだが、称号の現行しきい値（発見4件）は早すぎるため、牧野家の初登場だけは**称号ロジックと切り離した独立条件**で判定する。
- ホームに「殿の御使者」通知（巻物アイコン＋既存toast/効果音）→タップで一枚絵の会話（既存カード獲得演出フォーマット流用）。殿が「集めたご当地の品を献上せよ。励みによってはまだ知らぬ場所を教えてやろう」と誘う。
- ここで初めて図鑑内に「御殿」タブが出現（バッジで気づかせる）。
- **キャラは全て創作**。史実人物名は由緒解説の地の文のみに使い、喋らせない。

## 3. 年貢（納品）システム
### 何を納めるか
- おみやげ12種のうち**ダブり分（save.omiyage[id] >= 2）だけ納品可**。図鑑コンプ表示のため最低1個は必ず残す。
- **通常納品**: 好きなダブりを1〜3個献上→favor少量＋コイン少量。いつでも可の「賽銭」的行為（favor換算は小さくインフレ防止、例 1個=favor2）。
- **お達し（注文書, Phase2〜）**: 殿が特定おみやげを指名（未解禁エリア由来の品を欲しがる体で未踏エリアへ誘導）。達成でfavor大（例30〜50）＋既存アイテム（ダブルコイン券等）報酬。1日1件。

### テンポ／経済
- 通常納品はコスト0（アイテム消費のみ）・回数無制限だがfavor小。お達しは1日1件（ログボ/無料ガチャと同じ日次リズム）。
- ガチャ排出重み（おみやげ45%）は据え置き。**年貢は「出た後の使い道」を足すだけでガチャの回し得判定は変えない**。
- 解禁ペース目安: 1日1〜2回の通常納品＋週2〜3回のお達しで**1〜2週間で次の解禁**。

### 見返り＝解禁（新スポットの捻出案・要ファクトチェック）
現状32スポットは全開放済み → **牧野家経由でだけ解禁される隠しスポット**を新設して図鑑拡張。

| 案 | 実装 | 内容 |
|---|---|---|
| **蒼柴神社**（悠久山公園内） | 軽 | 3代藩主 牧野忠辰が藩祖 牧野忠成を祀り創建。最初の解禁先に最適（＝牧野家の始まり） |
| **蔵王堂城跡** | 軽〜中 | 牧野忠成が長岡城築城前に入った城。「殿がまだ長岡城を持つ前」の導入編向き |

## 4. 牧野家キャラと育成/好感度
全キャラ創作明記（特定の代の藩主や河井継之助本人を喋らせない）。
- **殿**（牧野の殿）: 好感度で表情/仕草差分（仏頂面→ニコニコ→大喜び）。
- **姫**: 年貢の受け取り窓口・図鑑解説ナビ。
- **家老**: 完全オリジナル。家臣ランク進行役・お達し取次ぎ。

**育成感の出し方（favor5段階）**: ①御殿背景が賑わう（庭に花・提灯・家臣の人影が増える）②殿の表情差分 ③フレーバーテキスト変化。画風は温かい和風水彩に統一（ChatGPT生成→透過処理パイプライン流用）。

## 5. 家臣団ランク（出世）— 探検家称号(TITLES)と併存の二軸
理由: TITLESは発見数ベースで実績と密結合。favorベースの新ランクは独立変数にし、二軸を別々に伸ばせる方がコンプ済みにも伸びしろを出せる。

| # | 呼称 |
|---|---|
| 1 | 中間（ちゅうげん） |
| 2 | 足軽 |
| 3 | 徒士（かち） |
| 4 | 平士（馬廻役） |
| 5 | 番頭（ばんがしら） |
| 6 | 用人 |
| 7 | 家老（藩の最高職＝到達点） |
| 特別 | 殿の御意見番（創作の名誉称号） |

**★史実訂正**: masa案の「老中」は藩士の役職ではなく**将軍直属の幕府職で藩主本人（大名）が就くもの**。家臣の出世双六の頂点に置くと史実誤用になるため、最高位は実在の藩職「家老」に留め、その上は**創作と明示した名誉称号**にする。

## 6. UI/導線
- **bottom-navタブは増やさない**（既に6項目）。花火館と同じく**図鑑の book-subtabs に「御殿」を追加**（`data-sub="makino"`）。
- 御殿タブ: ①上部=殿(＋姫/家老)＋好感度ゲージ＋家臣ランク名 ②中央=「年貢を納める」→omiyage-grid流用モーダルでダブり選択 ③お達しがある日はバッジ＋専用カード ④下部=次の解禁対象（伏せ字/シルエット）＋あとfavorいくつでプログレスバー。
- 演出: ランクアップ/解禁は既存 `toast()`＋`launchFireworks()`/`playFireworkShow()` 流用。解禁時マップにピンがポンと出る演出追加。ホームは「御用がある」時だけ小バッジ。

## 7. データ設計スケッチ（localStorage）
```js
save.makino = {
  seenIntro: false,      // 登場イベント既読
  favor: 0,              // 好感度累計
  rank: 0,               // 家臣ランクindex（0=中間 … 6=家老, 7=御意見番）
  tributesTotal: 0,      // 納品総数（実績用）
  tributes: {},          // omiyageId -> 納品累計
  unlockedSpots: [],     // 本システム経由で解禁したpoiId
  order: { need:{}, issuedDate:null, claimed:false },  // お達し
  favorRankSeen: 0       // ランクアップ演出の多重防止
};
```
- 登場条件はTITLES非依存の独立判定（例 found.length>=10）。
- 新スポットは `pois.json` に `unlockVia:"makino", unlockFavor:N` を追加。表示ロジックは「unlockViaが無ければ従来通り常時表示、あればfavor条件を満たすまで非表示」の分岐1箇所。
- ACHIEVEMENTSに makino_first / makino_karo / makino_favor_max 等を追加。TITLESは変更なし。
