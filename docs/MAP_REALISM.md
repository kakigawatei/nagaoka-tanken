# 地図を「実際の地形」に寄せる構想 ＋ Codex指示文

## 目的
今のマップは地区ごとの手描きイラストで、ピンは「地区内の相対％」で置いているため地理的に正確ではない。
これを **見た目は探検録の手描きトーンのまま／配置は実際の地形に忠実** にして、ピンが本物の場所にちゃんと乗るようにする。

## 仕組み（ここがキモ）
正確さは「絵」と「アプリ」の2つを噛み合わせて出す：

1. **各地区に"正確な地理的範囲（bbox）"を1つ決める**（下の表。四隅の緯度経度）。
2. **その範囲の実際の地図を、探検録トーンで"位置を崩さず"描き直す**（Codex）。北が上・範囲いっぱい・縮尺そのまま。
3. **アプリはそのbboxに対して緯度経度→％を計算**（`project()`は既存のまま。地区bboxを"POIから自動計算"→"この確定bbox"に差し替えるだけ）。

→ 3が揃うと、実座標のピンが「絵の中の正しい場所」に乗る。絵が実地形に忠実なほどピンが自然に見える。

## 大事な前提（期待値）
- Codex（画像生成）は**厳密なトレースはできず近似**。なので「実地形そっくりの雰囲気」にはなるが測量レベルの精度ではない。実地図のスクショを一緒に貼って「これを同じ配置で描き直して」と指示すると精度が上がる。
- **中心部(chuo)は正直な地理にすると中央の7スポットが密集する**（アオーレ〜河井記念館などは実際に半径300m内、悠久山だけ南東に離れている）。これは"本物の距離感"なので正しい姿。もし窮屈なら後日「中心部ズームのインセット地図」を足す手もある。
- 完全な精度が要るなら将来Leaflet＋実地図タイルという手もあるが、手描きの世界観が崩れる＆オンライン必須になるので、まずはこの"手描き×実地形"方式を推奨。

## 優先順位
- **最優先：chuo（8スポット）・settaya（4スポット）** ＝複数ピンで配置精度が効く。
- 単独スポットの地区（山古志/寺泊/栃尾/与板/越路）は実質1ピンの"風景絵"なので、realismは任意（今のままでも可、揃えたいなら同じ手順）。

## Codex指示文（テンプレート）
実地図のスクショをCodex/ChatGPTに貼って、下を送る（【】を各地区の値に置換）：
```
添付した実際の地図（【地区名】周辺）を、ゲーム「ながおか探検録」の世界観で描き直してください。

■ 画風：古い探検記・旅の記録帳のような、手描き＋水彩＋クラフト紙の質感。温かみのある配色（セピア・深緑・臙脂・金）。レトロで上品な絵本風。既存のゲーム内マップと同じトーン。

■ いちばん大事なルール（配置を崩さない）：
・北を上（north-up）にする。
・添付地図に写っている道路・信濃川などの川や水面・線路・海岸線・主要な建物や公園の「位置・向き・大きさの比率」を、そのままの配置で描くこと。位置を動かしたり、見栄えよく並べ替えたりしない。
・添付地図と同じ四角い範囲を、キャンバスの端まで（edge-to-edge）埋める。余白を作らない。
・縦横比は 【縦横比】（例：ほぼ正方形／縦長）にする。

■ 描き方：道は柔らかい線で、川や池は水彩で、公園や田畑は緑で、建物はシンプルな塊で。全体が地図として読み取れるように。
■ 文字ラベル・地名・ピン・マーカーは入れないでください（目印はアプリ側で後から重ねます）。
■ 高解像度、【推奨px】程度。

これはピンを重ねる"下地の地図"なので、実際の地形の配置に忠実であることを最優先にしてください。
```

## 納品・アプリ反映
- ファイル名：既存と同じ `map_<地区id>.png`（例 map_chuo.png / map_settaya.png）。差し替え。
- Drive「（新規）13_地図リメイク」等に納品→こちらでDL・軽量化・差し替え。
- **アプリ側の作業（こちらで実施）**：地区bboxを「POI自動計算(computeBBox)」から、上の表の**確定bbox(MAP_GEO)**に差し替え、その地区は確定bboxでピン投影。届いた地区から順に対応（未対応地区は今の自動計算のままフォールバック）。sw.jsのCACHE_VERSIONも上げる。
- 差し替え後、ヘッドレスで主要ピンが正しい位置に乗るか検証。

## 追記：既存マップの「色味だけ」を揃えるリメイク（2026-07-10）

### 背景
寺泊(teradomari)だけ色味が他地区と合っていなかった。原因は**彩度と色調の差**：
- 寺泊＝明るく高彩度・現代的（鮮やかな水色の海／黄色い道／ビビッドな緑／白い陸地）。
- 他地区（例：山古志）＝くすんだアンティーク水彩＋クラフト紙のセピア調（オリーブ〜セージの緑／ダスティブルーの水／生成りの道）。

### 方針
配置（海岸線・道・建物）を動かすとピン投影がズレるので、**geometryは維持したまま"色味・質感だけ"を既存の他マップに合わせて塗り直す**。フル描き直しではなく「リカラー」。

### Codex指示文テンプレ（色味リメイク用）
Codex（画像生成）に**2枚添付**：①直したい地図（例 `map_teradomari.png`＝配置の元）②手本にする既存マップ（例 `map_yamakoshi.png`＝色味の見本）。下を貼る（【】を置換）：
```
添付画像①（【地区名】の地図）の「地形・道路・海岸線・港・建物の配置」は一切変えずに、色味と質感だけを、添付画像②（手本のマップ）と同じトーンに描き直してください。目的は他の地区マップと色調を統一することです。

■ 変えないこと（超重要・配置を動かさない）
・海岸線・港・防波堤・道路・路地・建物の位置/大きさ/向きを添付①のまま完全維持。
・北を上（north-up）。キャンバス端まで（edge-to-edge）、余白なし。
・縦横比 約【縦横比】。出力 約【推奨px】。
・文字・地名・ピン・マーカーは入れない。

■ 色味・質感（添付②に合わせる）
・全体を「古い探検記／旅の記録帳」の風合いに。彩度を落とし温かみのあるセピア寄りに。クラフト紙（生成りの紙）の質感が全体に透ける。
・海／水：鮮やかな水色をやめ、くすんだ青（ダスティブルー、少しグレーがかった水色・低彩度）に。
・緑（山/田畑）：ビビッドな緑ではなく、くすんだオリーブ／セージグリーンに。
・道路：黄色をやめ、生成り〜淡いベージュ（紙より少し濃いクリーム）に。
・建物：淡いセピアの塊。
・水彩のにじみと紙の粒状感を残し、②と並べて同じ絵師が描いたと分かるくらい統一する。

要は「①の地図を、②のパレットと質感で塗り直したもの」を作ってください。配置の忠実さ＞見栄え。
```
- 水の色がブレる場合は、水面を含む既存マップ（例 `map_chuo.png` の川）も一緒に添付するとよい。
- 寺泊の値：縦横比 約0.79（縦長）／推奨 1200×1515px。

### 反映（こちら側）
届いた画像を軽量化 → 同名 `map_<地区id>.png` に差し替え → `sw.js` の CACHE_VERSION を上げる → ヘッドレスで主要ピンが正しい位置に乗るか検証 → 公開。geometryを維持したリカラーなので bbox 変更は不要。
