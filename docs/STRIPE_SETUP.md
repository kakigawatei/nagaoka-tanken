# Stripe 決済セットアップ手順（加盟店 月額サブスク）

加盟店の月額課金を、Stripeの「Payment Link（決済リンク）」で回すための手順。
運用：**申請（Tallyフォーム・無料）→ 審査 → 承認メールにプラン別の決済リンクを貼る → 加盟店がカード登録・課金開始 → 掲載**。カード管理・毎月の自動課金・領収書・解約までStripe側が対応するので、こちら側にサーバー不要。

> 事業主体：個人事業主／屋号 ながおか探検録製作委員会／岡雅俊（[[LEGAL_DRAFTS]] と同じ情報）。

---

## 手順

### 1. Stripeアカウント作成・有効化
1. https://stripe.com で登録（メール：aburalabo@gmail.com 推奨）。
2. 事業タイプ＝**個人事業主（Individual / Sole proprietor）**、国＝日本。
3. 本人確認（氏名・住所・生年月日）＋**入金先の銀行口座**を登録 → アカウント有効化。
4. 「特定商取引法に基づく表記」ページのURLや事業者情報を、Stripeの「公開ビジネス情報」に設定（領収書やチェックアウトに表示される）。

### 2. 商品（プラン）を3つ作成
Stripeダッシュボード →「商品カタログ」→「商品を追加」で、プランごとに **月額のくり返し（recurring / monthly）価格** を作成：

| 商品名 | 価格（税別） | 課金 |
|---|---|---|
| ながおか探検録 加盟店ライト | ¥3,000 | 毎月 |
| ながおか探検録 加盟店スタンダード | ¥6,000 | 毎月 |
| ながおか探検録 加盟店プレミアム | ¥12,000 | 毎月 |

- 「価格」を作るとき、請求期間＝**月次**、通貨＝JPY。
- 消費税：**税込にするか税別＋別途課税か**を決める。かんたんなのは「表示価格を税込」にしてしまう方式（例：ライトを税込¥3,300にする）。きちんとやるなら **Stripe Tax** を有効化して自動計算。

### 3. 決済リンク（Payment Link）を3つ発行
ダッシュボード →「Payment Links」→「新規作成」→ 各プランの月額価格を選択 → 作成。
- 数量変更を不可に（1件のみ）。
- 必要なら「請求先住所・事業者名を収集」をON。
- できた3本のURLを控える（承認メールに貼る／プラン紹介ページに載せる）。

### 4. 顧客ポータル（自己解約・カード変更）を有効化
「設定 → Billing → カスタマーポータル」を有効化。
- 加盟店が自分で**解約・カード更新・領収書ダウンロード**できるようになる（運営の手間が減る）。
- ポータルのURLを承認メールや掲載完了メールに添えておく。

### 5. メール・領収書
- 「設定 → メール」で、支払い成功・返金・請求書のメール送信をON（加盟店に自動で領収書が届く）。

---

## 運用フロー（1件ごと）
1. Tallyフォームで申請を受信（メール通知）。
2. 内容を審査 → 承認する場合、そのプランの**決済リンク**を承認メールに貼って送付。
3. 加盟店がリンクからカード登録 → サブスク開始（Stripeダッシュボードで入金・購読を確認）。
4. こちらで店舗ピン／カードを制作 → `shops.json` に追加 → デプロイで掲載。
5. 掲載完了メール（＋スタッフ用PIN＝クーポン確認用、＋顧客ポータルURL）を送付。

## 注意・メモ
- **手数料**：日本のカード決済はおおむね3.6%前後（最新はStripeの料金ページで確認）。プラン価格設定時に見込んでおく。
- **消費税**：課税事業者かどうか、税込/税別、インボイス（現在は登録なし）を踏まえて設定。将来インボイス登録したら領収書に登録番号を反映。
- **まずテストモード**で1件通してみる（テストカード 4242 4242 4242 4242／任意の未来の有効期限・任意のCVC）。問題なければ本番モードへ。
- **特商法・規約**：チェックアウトや承認メールから、利用規約・特商法表記（[[LEGAL_DRAFTS]]）にアクセスできるようにしておく。
- 将来フェーズB（自前フォーム→Stripe Checkout→webhookで自動掲載）に進むと、申請〜掲載を一気通貫で自動化できる（要バックエンド）。今はまず手動運用でOK。
