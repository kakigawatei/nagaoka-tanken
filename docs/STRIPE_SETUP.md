## 手順

### 1. Stripeアカウント作成・有効化
1. https://stripe.com で登録（メール：aburalabo@gmail.com 推奨）。
2. 事業タイプ＝**個人事業主（Individual / Sole proprietor）**、国＝日本。
3. 本人確認（氏名・住所・生年月日）＋**入金先の銀行口座**を登録 → アカウント有効化。
4. 「特定商取引法に基づく表記」ページのURLや事業者情報を、Stripeの「公開ビジネス情報」に設定（領収書やチェックアウトに表示される）。

### 2. 商品（プラン）を3つ作成
Stripeダッシュボード →「商品カタログ」→「商品を追加」で、プランごとに **月額のくり返し（recurring / monthly）価格** を作成：

| 商品名 | 価格（**税込**） | 課金 |
|---|---|---|
| ながおか探検録 加盟店ライト | ¥3,000 | 毎月 |
| ながおか探検録 加盟店スタンダード | ¥6,000 | 毎月 |
| ながおか探検録 加盟店プレミアム | ¥12,000 | 毎月 |
| ながおか探検録 初期登録料 | ¥5,000 | **1回のみ**（継続なし） |

- 「価格」を作るとき、請求期間＝**月次**、通貨＝JPY。初期登録料だけは「**一括（一回限り）**」で作る。
- **消費税＝税込総額表示に確定（2026-07-14 masaさん指示）**。上の金額が加盟店の支払総額そのもの。Stripeでは**税別＋別途課税にしない**（Stripe Taxは有効化しない／有効化する場合は「価格に税を含む」設定にする）。加盟店向け資料も全ページ「表示価格はすべて税込」と明記済み。
- 初回請求だけ「初期登録料¥5,000＋月額」の合算になる。Payment Linkは**月額価格＋初期登録料の2品を1本のリンクにまとめる**（Stripeは1リンク内に「継続」と「一回限り」を混在できる）と、初回=8,000円/2か月目以降=3,000円…のように自動で処理される。

### 3. 決済リンク（Payment Link）を3つ発行
ダッシュボード →「Payment Links」→「新規作成」→ 各プランの月額価格を選択 → 作成。
- 数量変更を不可に（1件のみ）。
- 必要なら「請求先住所・事業者名を収集」をON。
- できた3本のURLを控える（承認メールに貼る／プラン紹介ページに載せる）。

### 3.5 初期登録料の「先行加盟店キャンペーン」割引 ＝ Stripeのクーポンで自動化（2026-07-14 masaさん指示）

初期登録料（¥5,000）を**加盟順に割引**する：1〜10店舗目=100%OFF（無料）／11〜30店舗目=50%OFF（¥2,500）／31〜50店舗目=30%OFF（¥3,500）／51店舗目以降=通常。

**プラン別にリンクを量産しない**でよい。「商品カタログ → クーポン」で3つ作り、**適用対象商品を「初期登録料」だけに限定**する（`applies_to`）：

| クーポン名（コード） | 割引 | 適用上限（max redemptions） |
|---|---|---|
| OPENING100 | 100%OFF | **10回** |
| FOUNDER50 | 50%OFF | **20回**（11〜30店舗目ぶん） |
| EARLY30 | 30%OFF | **20回**（31〜50店舗目ぶん） |

- **`利用上限回数（max_redemptions）` を設定すれば、枠が埋まると自動で使えなくなる**ので、店舗数のカウントを手で管理しなくていい。上のコードを使い切ったら次のコードを承認メールに載せる運用。
- Payment Link 作成時に「**プロモーションコードの入力を許可**」をONにして、承認メールに「クーポンコード：OPENING100」と書けばOK。
- 割引は**初期登録料のみ**に効く（月額には効かない）ように、クーポンの適用対象商品を初期登録料に限定すること。

### 4. 顧客ポータル（自己解約・カード変更）を有効化 ※**解約はOFFにする**
「設定 → Billing → カスタマーポータル」を有効化。
- **⚠️ 最低利用期間3か月（2026-07-14 masaさん決定）を守らせるため、ポータルの「サブスクリプションのキャンセルを許可」は OFF にする**。ONのままだと加盟店が1か月目でも自分で解約できてしまい、規約の3か月縛りが実質機能しない。
- **解約はメール申請のみ**受付 → 運営がStripeダッシュボードで期間満了時キャンセル（`cancel_at_period_end`）を設定する運用。
- ポータルでは**カード更新・領収書ダウンロード**だけ許可（この2つは加盟店にやってもらった方が運営が楽）。
- ポータルのURLを承認メールや掲載完了メールに添えておく。

> Stripe単体では「最低3か月」を自動強制できない（Payment Linkの月額サブスクは毎月更新のため）。**規約（[[LEGAL_DRAFTS]] 第8条）で縛り、ポータルの自己解約をOFFにする**のが実務的な担保。もっと固くしたいなら「3か月分を初回一括前払い」にする手もあるが、加盟のハードルが上がるので今は非推奨。

### 5. メール・領収書
- 「設定 → メール」で、支払い成功・返金・請求書のメール送信をON（加盟店に自動で領収書が届く）。

## 銀行振込（カードが使えない／使いたくないお店向け）※2026-07-14 masaさん決定

カード決済（Stripe）と**並行して銀行振込も受ける**。ただし**毎月の振込＋毎月の入金確認は事務が破綻する**ので、**まとめ前払い**に限定する。

- **3か月分の一括前払いが基本**（最低利用期間3か月と一致するので説明もキレイ）。
- **12か月分の一括なら1か月分サービス**（＝11か月分の料金で12か月掲載）。入金が前倒しで入り、事務も年1回で済む。
- 初回請求＝**初期登録料（キャンペーン割引後）＋3か月分 or 12か月分**。
- **振込手数料は加盟店負担**。振込人名義は店舗名でお願いする（入金消込のため）。
- **入金確認後に掲載開始**（未入金なら掲載しない＝取りっぱぐれなし）。支払期限は請求書発行から14日。
- 更新は期限1か月前に次の請求書を送付。未入金なら掲載停止。
- **請求書には「適格請求書（インボイス）ではありません」と明記**（免税事業者・登録番号なしのため）。書いておかないと相手の経理でモメる。

### 請求書PDFの作り方（自動化済み）
```
node docs/merchant/make_invoice.js --shop "○○食堂" --plan standard --months 3 --init 0
```
→ `docs/merchant/out/請求書_○○食堂_NT-….pdf` が出る（金額・期限・振込先・注記は自動）。`--months 12` なら1か月分サービスを自動で差し引く。`--init` はキャンペーン割引後の初期登録料（0 / 2500 / 3500 / 5000）。

> 🔒 **口座情報は `docs/merchant/bank.local.json` にだけ書く。このファイルは `.gitignore` 済み**。**このリポジトリは公開（GitHub Pages）なので、口座番号を資料PDFやドキュメントに書くと全世界に公開される**。振込先は請求書PDFと承認メールにだけ載せること。
> 💡 楽天銀行は個人事業主向けの**屋号付き口座**が作れる。名義が「ながおか探検録製作委員会（オカ マサトシ）」になると法人相手の信用が上がる（個人名義だと経理を通しにくい会社がある）。

## 運用フロー（1件ごと）
1. Tallyフォームで申請を受信（メール通知）。
2. 内容を審査 → 承認する場合、そのプランの**決済リンク**を承認メールに貼って送付。
3. 加盟店がリンクからカード登録 → サブスク開始（Stripeダッシュボードで入金・購読を確認）。
4. こちらで店舗ピン／カードを制作 → `shops.json` に追加 → デプロイで掲載。
5. 掲載完了メール（＋スタッフ用PIN＝クーポン確認用、＋顧客ポータルURL）を送付。
